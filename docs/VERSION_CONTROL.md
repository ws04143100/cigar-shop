# 文檔版本控制規範

## 版本號格式
`v[主要版本].[次要版本].[修訂版本] - YYYY-MM-DD`

範例：`v1.0.0 - 2026-03-23`

## 變更類型

| 類型 | 說明 |
|------|------|
| MAJOR | 重大結構變更 |
| MINOR | 新增功能 |
| PATCH | 錯誤修正 |

## 需要版本控制的文檔

1. `docs/README.md` - 項目說明
2. `docs/DEPLOY.md` - 部署指南
3. `docs/SOP.md` - 標準作業流程
4. `docs/CHANGELOG.md` - 變更日誌

## 變更日誌格式

```markdown
## [v1.1.0] - 2026-03-23

### ✨ 新增
- Excel 一鍵更新腳本

### 🔧 優化
- 記憶系統標籤化

### 🐛 修復
- 修復批發表檔名錯誤
```

## 更新頻率
- 每次功能上線 → 更新 CHANGELOG
- 每月 → 回顧並更新版本號
